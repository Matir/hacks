# CWE-416: Use After Free

### Description
A **Use After Free (UAF)** vulnerability occurs when an application continues to use a pointer after the memory area it points to has been deallocated (freed). In memory management, when `free()` (in C) or `delete` (in C++) is called, the memory is returned to the system or the allocator's pool. However, the pointer variable itself often retains the address of that memory. If the program later dereferences that "dangling pointer," it may access memory that has been reallocated for a different purpose.

This vulnerability is a cornerstone of modern exploitation, particularly in complex software written in unmanaged languages like C and C++.

### Impact
The consequences of a Use After Free vulnerability range from instability to full system compromise:
*   **Denial of Service (DoS):** Dereferencing a dangling pointer often causes the application to crash due to segmentation faults.
*   **Information Disclosure:** An attacker may be able to read sensitive data if the freed memory has been reallocated to store different information.
*   **Arbitrary Code Execution (ACE/RCE):** This is the most severe impact. If an attacker can control the contents of the reallocated memory (e.g., by triggering specific allocations), they can overwrite function pointers or virtual method tables (vtable). When the program later "uses" the dangling pointer, it may execute attacker-controlled code.

### Remediation
Preventing Use After Free requires disciplined memory management:

1.  **Nullify Pointers After Freeing:** Immediately set a pointer to `NULL` (or `nullptr`) after calling `free()` or `delete`. This ensures that any subsequent use of the pointer results in a predictable null-pointer crash rather than a silent corruption or exploitable state.
2.  **Use Smart Pointers (C++):** In C++, avoid manual calls to `new` and `delete`. Use RAII (Resource Acquisition Is Initialization) via `std::unique_ptr` or `std::shared_ptr`. These manage memory automatically and significantly reduce the risk of UAF.
3.  **Scope Management:** Minimize the scope of pointers. If a pointer is only needed within a small block, ensure it cannot be accessed outside that block.
4.  **Memory Sanitizers:** During development and testing, use tools like **AddressSanitizer (ASan)**. These tools instrument the code to detect UAF and other memory errors at runtime.
5.  **Secure Allocation Libraries:** Consider using memory allocators that include hardening features, such as "quarantine" zones for recently freed memory.

### Examples

#### Vulnerable Code (C)
In this example, the pointer `data` is used to log a message after it has been freed.
```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void process_request(char *input) {
    char *data = (char *)malloc(100);
    strncpy(data, input, 99);
    
    // ... logic ...

    if (strlen(data) > 50) {
        free(data); // Memory is freed here
    }

    // VULNERABLE: Use after free if data was > 50 chars
    printf("Processing: %s\n", data); 
}
```

#### Secure Code (C)
By nullifying the pointer and adding a check, the vulnerability is mitigated.
```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void process_request(char *input) {
    char *data = (char *)malloc(100);
    if (!data) return;
    strncpy(data, input, 99);
    
    if (strlen(data) > 50) {
        free(data);
        data = NULL; // Pointer is nullified
    }

    // SECURE: Pointer check prevents use after free
    if (data != NULL) {
        printf("Processing: %s\n", data);
    } else {
        printf("Processing: Data exceeded length limit and was cleared.\n");
    }
}
```

#### Secure Code (C++ Smart Pointers)
The modern C++ approach eliminates manual memory management entirely.
```cpp
#include <iostream>
#include <memory>
#include <string>

void process_request(std::string input) {
    // Use unique_ptr for automatic memory management
    auto data = std::make_unique<std::string>(input);

    if (data->length() > 50) {
        data.reset(); // Safely destroys the object and sets pointer to null
    }

    if (data) {
        std::cout << "Processing: " << *data << std::endl;
    }
}
```
