# CWE-476: NULL Pointer Dereference

## Description
A NULL pointer dereference occurs when an application attempts to access memory using a pointer that is expected to be valid but is actually `NULL`. In many programming languages, `NULL` is a special value indicating that the pointer or reference does not point to any valid object or memory address. 

When the software dereferences this pointer—meaning it tries to read from or write to the memory address the pointer points to—it triggers a runtime exception or a system crash. In C and C++, this is considered "Undefined Behavior" and typically results in a segmentation fault (SIGSEGV). In managed languages like Java or C#, it results in a `NullPointerException` or `NullReferenceException`.

This vulnerability often arises from:
1.  Failure to check the return value of functions that can return NULL (e.g., `malloc`, `fopen`, `lookup`).
2.  Complex conditional logic where a pointer is checked for NULL in one path but used in another.
3.  Race conditions where a pointer is invalidated by another thread between the check and the use (TOCTOU).

## Impact
The most common impact of a NULL pointer dereference is a **Denial of Service (DoS)**. By causing the application to crash, an attacker can disrupt the availability of a service. 

In specific, rare contexts—such as kernel-mode execution or systems where the NULL page (address 0) is mapable—an attacker might be able to exploit a NULL pointer dereference to execute arbitrary code or bypass security checks. However, in modern user-mode applications on protected operating systems, the impact is almost exclusively limited to process termination.

## Remediation
The primary defense against NULL pointer dereferences is rigorous input validation and defensive programming.

1.  **Check Return Values:** Always verify that functions returning pointers/references (like memory allocators or database lookups) succeeded before using the result.
2.  **Defensive Checks:** Perform "Sanity Checks" or "Guard Clauses" at the beginning of functions to ensure parameters are not NULL.
3.  **Modern Language Features:**
    *   **Java:** Use `Optional<T>` to represent values that might be missing.
    *   **C#:** Utilize Nullable Reference Types and the null-conditional operator (`?.`).
    *   **C++:** Use smart pointers (`std::unique_ptr`, `std::shared_ptr`) and references instead of raw pointers where possible.
4.  **Static Analysis:** Use static analysis tools (like Semgrep, SonarQube, or Clang Static Analyzer) during development to identify paths where a NULL dereference could occur.
5.  **Compile-time Assertions:** Use attributes like `__attribute__((nonnull))` in C/C++ to help the compiler catch errors.

## Examples

### C (Unchecked Memory Allocation)

**Vulnerable:**
The following code does not check if `malloc` fails. If the system is out of memory, `malloc` returns `NULL`, and the assignment `*ptr = 10` will cause a crash.

```c
void vulnerable_function() {
    int *ptr = (int *)malloc(sizeof(int));
    // Missing NULL check
    *ptr = 10; 
    printf("%d", *ptr);
    free(ptr);
}
```

**Secure:**
The code checks the pointer before dereferencing it.

```c
void secure_function() {
    int *ptr = (int *)malloc(sizeof(int));
    if (ptr == NULL) {
        // Handle allocation failure
        return;
    }
    *ptr = 10;
    printf("%d", *ptr);
    free(ptr);
}
```

### Java (Logic Error)

**Vulnerable:**
In this case, the developer checks for `null` but then continues to use the object inside the block where it is known to be `null`, or fails to return/exit after the check.

```java
public void processOrder(Order order) {
    if (order == null) {
        logger.error("Order is null!");
    }
    // If order was null, the next line throws NullPointerException
    System.out.println(order.getId());
}
```

**Secure:**
The method returns immediately if the object is null, or uses an `else` block.

```java
public void processOrder(Order order) {
    if (order == null) {
        logger.error("Order is null!");
        return; // Stop execution
    }
    System.out.println(order.getId());
}
```
