### Description

The "Improper Restriction of XML External Entity Reference" vulnerability (XXE) arises when an XML parser is configured to process external entities and the application fails to properly sanitize or restrict the XML input. XML documents can define external entities, which are references to external resources (files, URLs). If an attacker can control the XML input and the application processes these external entities, they can potentially read arbitrary files on the server, execute arbitrary code (in some cases), conduct denial-of-service attacks, or perform server-side request forgery (SSRF).

The fundamental problem lies in the XML parser's default or misconfigured behavior, allowing it to resolve external entities without proper validation or restriction. This can be exploited by crafting malicious XML documents that point to sensitive local files or external URLs.

### Impact

Successful exploitation of an XXE vulnerability can have severe consequences:

*   **Data Leakage:** An attacker can read sensitive files from the server, such as configuration files, password files, or source code.
*   **Remote Code Execution (RCE):** In some configurations (e.g., using `expect` in PHP), an attacker can execute arbitrary code on the server.
*   **Denial of Service (DoS):** An attacker can cause the server to consume excessive resources by referencing extremely large or recursively defined external entities, leading to a denial of service.
*   **Server-Side Request Forgery (SSRF):** An attacker can use the server to make requests to internal or external systems, potentially accessing resources that are normally protected by firewalls or other network controls.

### Remediation

The most effective way to prevent XXE vulnerabilities is to disable the processing of external entities altogether. If external entities are genuinely required, then extremely careful validation and sanitization of the XML input are necessary, but strongly discouraged.

**General Recommendations:**

*   **Disable External Entities:** Disable the processing of external entities in the XML parser configuration.
*   **Input Validation:** If disabling external entities is not possible, carefully validate and sanitize the XML input to ensure that it does not contain any malicious external entity references. Use a whitelist approach to restrict the allowed values and data types.
*   **Least Privilege:** Run the application with the least privileges necessary.
*   **Update Libraries:** Keep XML processing libraries up to date to benefit from the latest security patches.
*   **Use a SAST tool:** Static analysis tools will detect XXE vulnerabilities.

**Language-Specific Examples:**

*   **Java:**
    ```java
    import javax.xml.parsers.SAXParserFactory;
    import javax.xml.parsers.SAXParser;
    import org.xml.sax.XMLReader;

    // Vulnerable Code
    // SAXParserFactory factory = SAXParserFactory.newInstance();
    // SAXParser parser = factory.newSAXParser();

    // Secure Code
    SAXParserFactory factory = SAXParserFactory.newInstance();
    factory.setFeature("http://xml.org/sax/features/external-general-entities", false);
    factory.setFeature("http://xml.org/sax/features/external-parameter-entities", false);
    factory.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true); //Prevent DoS
    SAXParser parser = factory.newSAXParser();
    XMLReader reader = parser.getXMLReader();

    // Further processing with the 'reader' object
    ```

*   **Python:**
    Prefer using `defusedxml` instead of `xml.etree.ElementTree` or `xml.sax`. If using `xml.etree.ElementTree` you must mitigate XXE by either removing the `XML` support to parse from string by setting the parser to be something else than the default `XMLParser` or overriding resolve_entities.

    ```python
    # Vulnerable Code
    # import xml.etree.ElementTree as ET
    # root = ET.fromstring(xml_data)

    # Secure Code (using defusedxml)
    import defusedxml.ElementTree as ET
    root = ET.fromstring(xml_data)

    # Secure Code (alternative using ElementTree)
    import xml.etree.ElementTree as ET

    parser = ET.XMLParser(resolve_entities=False)
    root = ET.fromstring(xml_data, parser=parser)
    ```

*   **PHP:**
    ```php
    // Vulnerable Code
    // $xml = simplexml_load_string($xmlString, 'SimpleXMLElement', LIBXML_NOENT);

    // Secure Code
    libxml_disable_entity_loader(true);
    $xml = simplexml_load_string($xmlString);
    ```

### Examples

**Vulnerable Code Example (Java):**

```java
import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import org.w3c.dom.Document;
import java.io.ByteArrayInputStream;

public class XXEVulnerable {
    public static void main(String[] args) throws Exception {
        String xmlString = "<!DOCTYPE foo [ <!ENTITY xxe SYSTEM \"file:///etc/passwd\"> ]><foo>&xxe;</foo>";

        DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
        DocumentBuilder builder = factory.newDocumentBuilder();
        Document doc = builder.parse(new ByteArrayInputStream(xmlString.getBytes()));

        System.out.println("Parsed document: " + doc.getDocumentElement().getTextContent());
    }
}
```

**Secure Code Example (Java):**

```java
import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import org.w3c.dom.Document;
import java.io.ByteArrayInputStream;

public class XXESecure {
    public static void main(String[] args) throws Exception {
        String xmlString = "<!DOCTYPE foo [ <!ENTITY xxe SYSTEM \"file:///etc/passwd\"> ]><foo>&xxe;</foo>";

        DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
        factory.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
        factory.setFeature("http://xml.org/sax/features/external-general-entities", false);
        factory.setFeature("http://xml.org/sax/features/external-parameter-entities", false);
        DocumentBuilder builder = factory.newDocumentBuilder();

        Document doc = builder.parse(new ByteArrayInputStream(xmlString.getBytes()));

        System.out.println("Parsed document: " + doc.getDocumentElement().getTextContent());
    }
}
```
