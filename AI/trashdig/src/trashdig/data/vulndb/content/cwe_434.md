# CWE-434: Unrestricted Upload of File with Dangerous Type

## Description
CWE-434 occurs when an application allows a user to upload a file to the server's filesystem without sufficiently validating the file's type, name, or content. When an application accepts file uploads, it must ensure that the file cannot be executed as code by the server or used to perform path traversal attacks. 

The most common manifestation of this vulnerability is the "Webshell" attack, where an attacker uploads a script file (e.g., `.php`, `.jsp`, `.asp`, `.py`) to a directory accessible by the web server. If the server is configured to execute these scripts, the attacker can gain Remote Code Execution (RCE) on the underlying system.

## Impact
*   **Remote Code Execution (RCE):** Attackers can execute arbitrary commands on the server by uploading and calling a malicious script.
*   **Full System Compromise:** Once RCE is achieved, attackers can pivot to internal networks, access databases, or escalate privileges.
*   **Defacement:** Attackers can replace legitimate web pages with their own content.
*   **Malware Distribution:** The server can be used to host and distribute malware to other users.
*   **Denial of Service (DoS):** Attackers can upload extremely large files to exhaust disk space or inode limits.
*   **Cross-Site Scripting (XSS):** Uploading files with `.html` or `.svg` extensions containing malicious scripts.

## Remediation

### 1. Whitelist Allowed Extensions
Never use a "blacklist" (denying `.php`, `.exe`). Attackers can bypass blacklists using alternative extensions like `.php5`, `.phtml`, or `.phar`. Instead, explicitly define a whitelist of allowed extensions (e.g., `['jpg', 'jpeg', 'png', 'pdf']`).

### 2. Validate File Content (MIME Sniffing)
Do not trust the `Content-Type` header sent by the browser. Use server-side libraries to inspect the "magic numbers" (file signatures) of the file to verify its actual type.

### 3. Rename Uploaded Files
Do not use the user-provided filename. Generate a random, unique string (like a UUID) for the filename on the server. This prevents path traversal attacks (e.g., `../../etc/passwd`) and prevents attackers from predicting the URL of their uploaded file.

### 4. Store Files Outside the Web Root
Save uploaded files in a directory that is not directly accessible via the web server's document root. Serve them through a controlled script that sets the appropriate `Content-Disposition` headers.

### 5. Disable Execution Permissions
Configure the upload directory to prevent the execution of scripts. For example, in Apache, use `.htaccess` to disable engine processing, or mount the storage partition with the `noexec` flag.

## Examples

### Vulnerable Code (Python/Flask)
The following code accepts a file and saves it using the original filename in a public directory without any validation.

```python
from flask import Flask, request
import os

app = Flask(__name__)
UPLOAD_FOLDER = 'static/uploads'

@app.route('/upload', methods=['POST'])
def upload_file():
    if 'file' not in request.files:
        return "No file part"
    file = request.files['file']
    # VULNERABLE: Uses original filename and saves to a web-accessible directory
    file.save(os.path.join(UPLOAD_FOLDER, file.filename))
    return "File uploaded successfully"
```

### Secure Code (Python/Flask)
This version validates the extension against a whitelist, sanitizes the filename, and checks the file content type.

```python
import os
import uuid
import magic # python-magic library for MIME sniffing
from flask import Flask, request, abort
from werkzeug.utils import secure_filename

app = Flask(__name__)
UPLOAD_FOLDER = '/var/www/uploads_data/' # Outside web root
ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'gif'}

def allowed_file(filename, stream):
    # Check extension
    ext = filename.rsplit('.', 1)[1].lower() if '.' in filename else None
    if ext not in ALLOWED_EXTENSIONS:
        return False
    
    # Check MIME type (Magic Numbers)
    header = stream.read(2048)
    stream.seek(0) # Reset stream pointer
    mime = magic.from_buffer(header, mime=True)
    return mime in ['image/png', 'image/jpeg', 'image/gif']

@app.route('/upload', methods=['POST'])
def upload_file():
    if 'file' not in request.files:
        abort(400)
    
    file = request.files['file']
    
    if file and allowed_file(file.filename, file.stream):
        # Secure the name and assign a UUID
        filename = str(uuid.uuid4()) + "_" + secure_filename(file.filename)
        file.save(os.path.join(UPLOAD_FOLDER, filename))
        return "File uploaded securely"
    else:
        return "Invalid file type", 400
```
