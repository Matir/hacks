## Description

Server-Side Request Forgery (SSRF) is a vulnerability that allows an attacker to induce the server to make HTTP requests to arbitrary domains of the attacker's choosing. In typical SSRF attacks, the attacker can cause the server to connect to internal-only services within the organization's infrastructure. In other cases, they may be able to force the server to connect to arbitrary external systems, potentially leaking sensitive data such as authorization credentials. SSRF occurs when an application fetches a remote resource without validating the user-supplied URL. This allows the attacker to control the destination to which the server makes a request.

## Impact

The impact of SSRF can be severe, potentially leading to:

*   **Access to internal resources:** Attackers can bypass firewalls and access internal services or databases that are not directly accessible from the outside.
*   **Information disclosure:** Sensitive data stored on internal systems can be leaked to the attacker.
*   **Remote code execution:** In some cases, SSRF can be chained with other vulnerabilities (e.g., file upload, command injection) to achieve remote code execution on the server.
*   **Denial of service:** An attacker might be able to overload internal resources or external systems by making a large number of requests.
*   **Port scanning of internal networks:** Attackers can map out the internal network to identify other potential targets.
*   **Credential theft:** SSRF can be used to access metadata APIs (e.g., AWS EC2 metadata) and retrieve sensitive credentials.

## Remediation

Preventing SSRF requires a multi-layered approach:

1.  **Input Validation:**
    *   **Whitelist allowed URLs:** Ideally, only allow requests to a pre-defined set of URLs. This provides the strongest security.
    *   **Validate and sanitize user-supplied URLs:** If whitelisting is not feasible, carefully validate and sanitize user-supplied URLs.  Check for:
        *   Invalid or unexpected URL schemes (e.g., `file://`, `ftp://`, `gopher://`). Only allow `http://` and `https://` if necessary.
        *   Private IP addresses (e.g., `127.0.0.1`, `192.168.*.*`, `10.*.*.*`, `172.16.*.* - 172.31.*.*`).
        *   Hostname resolution to private IP addresses. Prevent the server from resolving hostnames to private IPs.
        *   Bypass attempts (e.g., using URL encoding, octal or hexadecimal IP representations).
2.  **Network Segmentation:**
    *   Isolate internal resources and services from the internet.
    *   Use firewalls and access control lists (ACLs) to restrict network access.
3.  **Authentication and Authorization:**
    *   Implement strong authentication and authorization mechanisms for internal resources.
    *   Ensure that services require authentication even from within the internal network.
4.  **Disable unused URL schemes:** Disable URL schemes that are not required (e.g., `file://`, `ftp://`, `gopher://`).
5.  **Use a centralized HTTP client:** Employ a centralized HTTP client library or service that enforces security policies and applies consistent validation.
6.  **Monitor and log outbound requests:** Monitor and log all outbound HTTP requests made by the server. This can help detect suspicious activity.

**Language-Specific Examples:**

*   **PHP:**

    **Vulnerable:**

    ```php
    <?php
    $url = $_GET['url'];
    $response = file_get_contents($url);
    echo $response;
    ?>
    ```

    **Secure:**

    ```php
    <?php
    $url = $_GET['url'];
    $allowed_hosts = ['example.com', 'api.example.com'];
    $parsed_url = parse_url($url);

    if (isset($parsed_url['host']) && in_array($parsed_url['host'], $allowed_hosts)) {
        $response = file_get_contents($url);
        echo $response;
    } else {
        echo "Invalid URL";
    }
    ?>
    ```

*   **Python:**

    **Vulnerable:**

    ```python
    import requests
    from flask import request

    url = request.args.get('url')
    response = requests.get(url)
    print(response.text)
    ```

    **Secure:**

    ```python
    import requests
    from flask import request

    allowed_hosts = ['example.com', 'api.example.com']
    url = request.args.get('url')
    try:
        parsed_url = urlparse(url)
        if parsed_url.netloc in allowed_hosts:
            response = requests.get(url)
            print(response.text)
        else:
            print("Invalid URL")
    except Exception as e:
        print("Invalid URL format")
    ```

*   **Java:**

    **Vulnerable:**

    ```java
    import java.net.*;
    import java.io.*;

    public class SSRFExample {
        public static void main(String[] args) throws Exception {
            String urlString = args[0]; // Get URL from command line
            URL url = new URL(urlString);
            InputStream in = url.openStream();
            BufferedReader reader = new BufferedReader(new InputStreamReader(in));
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
            in.close();
        }
    }
    ```

    **Secure:**

    ```java
    import java.net.*;
    import java.io.*;
    import java.util.Arrays;
    import java.util.List;

    public class SSRFExample {
        public static void main(String[] args) throws Exception {
            String urlString = args[0]; // Get URL from command line
            List<String> allowedHosts = Arrays.asList("example.com", "api.example.com");

            URL url = new URL(urlString);
            String host = url.getHost();

            if (allowedHosts.contains(host)) {
                InputStream in = url.openStream();
                BufferedReader reader = new BufferedReader(new InputStreamReader(in));
                String line;
                while ((line = reader.readLine()) != null) {
                    System.out.println(line);
                }
                in.close();
            } else {
                System.out.println("Invalid URL");
            }
        }
    }
    ```

## Examples

**Vulnerable Example (PHP):**

```php
<?php
$target_url = $_GET['url'];
$response = file_get_contents($target_url);
echo $response;
?>
```

An attacker can provide a URL like `?url=http://internal.server/secret.txt` to access internal files.

**Secure Example (PHP):**

```php
<?php
$target_url = $_GET['url'];
$allowed_hosts = array("www.example.com", "example.com");
$url_parts = parse_url($target_url);

if (isset($url_parts['host']) && in_array($url_parts['host'], $allowed_hosts)) {
  $response = file_get_contents($target_url);
  echo $response;
} else {
  echo "Invalid URL";
}
?>
```

This code validates the host of the target URL against a whitelist of allowed hosts. Only requests to allowed hosts are processed.
