## Description
Server-Side Request Forgery (SSRF) occurs when a web application fetches a remote resource without sufficiently validating the user-supplied URL. An attacker can use this functionality to force the application to send a crafted request to an unexpected destination, often bypassing security controls like firewalls or VPC isolation.

In a typical SSRF attack, the attacker targets:
1.  **Internal Services**: Accessing services running on `localhost` or the internal network (e.g., administrative panels, databases, or Redis instances) that are not exposed to the internet.
2.  **Cloud Metadata Services**: Querying the internal metadata APIs of cloud providers (e.g., AWS/GCP/Azure) at `169.254.169.254` to steal temporary credentials and IAM roles.
3.  **Out-of-Band (OOB) Exfiltration**: Using the server to perform port scanning or interact with external malicious servers to leak internal data.

## Impact
The impact of SSRF can range from information disclosure to full system compromise:
*   **Credential Theft**: Accessing cloud instance metadata often yields high-privilege API keys.
*   **Internal Network Mapping**: Identifying open ports and services on the internal infrastructure.
*   **Bypassing Authentication**: Accessing internal APIs that trust the server's IP address and do not require further authentication.
*   **Remote Code Execution (RCE)**: By interacting with internal services like Redis or Memcached, an attacker may be able to inject commands or modify configuration files.

## Remediation
The most effective way to prevent SSRF is to avoid building URLs from user input. If the application must fetch user-defined URLs, follow these steps:

1.  **Strict Allow-listing**: Maintain a list of approved hostnames or IP addresses. Do not allow users to specify arbitrary domains.
2.  **Validate the Scheme**: Only allow `http` or `https`. Explicitly block `file://`, `gopher://`, `ftp://`, etc.
3.  **DNS Resolution Validation**: Resolve the user-provided hostname to an IP address before making the request. Ensure the resolved IP is not a private, loopback, or reserved address (e.g., `127.0.0.1`, `10.0.0.0/8`, `169.254.169.254`).
4.  **Prevent DNS Rebinding**: Ensure that the IP address validated in step 3 is the same one used for the actual request (some libraries allow passing a pre-resolved IP).
5.  **Network Isolation**: Run the application in a network segment with restricted outbound access. Use a proxy for all external requests and monitor logs.

## Examples

### Vulnerable Example (Node.js/Express)
The following code accepts a `url` parameter and fetches it without validation. An attacker could provide `?url=http://169.254.169.254/latest/meta-data/` to access AWS metadata.

```javascript
const express = require('express');
const axios = require('axios');
const app = express();

app.get('/proxy', async (req, res) => {
    const targetUrl = req.query.url;
    
    // VULNERABLE: Direct use of user-supplied URL
    try {
        const response = await axios.get(targetUrl);
        res.send(response.data);
    } catch (error) {
        res.status(500).send('Error fetching resource');
    }
});
```

### Secure Example (Node.js/Express)
This example uses an allow-list to ensure only trusted domains are contacted.

```javascript
const express = require('express');
const axios = require('axios');
const { URL } = require('url');
const app = express();

const ALLOWED_HOSTS = ['api.trusted-partner.com', 'images.mysite.com'];

app.get('/proxy', async (req, res) => {
    const targetUrl = req.query.url;

    try {
        const parsedUrl = new URL(targetUrl);
        
        // SECURE: Validate scheme
        if (parsedUrl.protocol !== 'https:') {
            return res.status(400).send('Only HTTPS allowed');
        }

        // SECURE: Check against allow-list
        if (!ALLOWED_HOSTS.includes(parsedUrl.hostname)) {
            return res.status(403).send('Forbidden destination');
        }

        const response = await axios.get(targetUrl);
        res.send(response.data);
    } catch (error) {
        res.status(400).send('Invalid URL or error fetching resource');
    }
});
```
